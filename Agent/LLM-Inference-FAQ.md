## vLLM 篇

### 1. 什么是 vLLM？它解决了什么问题？

**Q:** vLLM 为什么比原始 Hugging Face 推理快很多？核心是什么技术？

**A:**

**vLLM** 是一个**开源大语言模型推理加速引擎**，主要解决大模型长文本推理时显存浪费和吞吐量低的问题。

它的核心创新是 **PagedAttention** 技术，解决了 KV 缓存碎片化问题。

**原生 Hugging Face 推理的问题：**
- 每个 sequence 都需要连续一块显存存 KV cache
- 不同 sequence 长度不同，显存碎片化严重
- 显存利用率低，吞吐量上不去

**vLLM 通过 PagedAttention 解决：**
- 把 KV cache 分成固定大小的"页"
- 逻辑连续，物理不连续，用页表映射
- 显存利用率从 ~50% 提升到 ~90%
- 能支持更多并发请求，吞吐量大幅提升

**简单说：vLLM 主要就是通过更好的 KV 缓存管理，把显存利用率提上去，从而提升吞吐量。**

---

### 2. 什么是 PagedAttention？原理是什么？

**Q:** 讲讲 PagedAttention 设计思路。

**A:**

PagedAttention 借鉴了操作系统虚拟内存分页的思想：

| 概念 | 对应 vLLM |
|------|----------|
| 虚拟内存页 | KV 块（固定大小）|
| 页表 | 块映射表（逻辑块号 → 物理块号）|
| 缺页中断 | 分配物理块 |
| 交换分区 | 把不用的块换出到 CPU 内存 |

**核心优点：**
1. **处理变长序列高效**：不用提前分配最大长度连续显存，按需分配
2. **显存利用率高**：解决碎片化，利用率从 ~50% → ~90%
3. **支持连续批处理（continuous batching）**：新来的请求可以随时插进去，不用等一批都处理完，提高了 GPU 利用率

---

### 3. vLLM 的 continuous batching（连续批处理）是什么？和静态批处理有什么区别？

**Q:** 什么是连续批处理？为什么快？

**A:**

**静态批处理（传统做法）**：
- 把多个请求攒成一批，一起处理
- 必须等这一批所有序列都处理完，才能处理下一批
- 如果序列长短不一，短序列处理完了也要等长序列，GPU 资源浪费

**连续批处理（vLLM 做法）**：
- 每个 step 都把新来的请求插进去一起处理
- 不用等整批完成，处理完一个就输出一个
- GPU 利用率更高，吞吐量更高，平均延迟更低

**对比：**

| 指标 | 静态批处理 | 连续批处理 |
|------|------------|-------------|
| GPU 利用率 | 低（长短不齐，等待浪费） | 高 |
| 吞吐量 | 低 | 高 |
| 平均延迟 | 高 | 低 |

---

### 4. vLLM 支持哪些量化方法？

**Q:** vLLM 能支持哪些量化方式来省显存？

**A:**

vLLM 主流支持：
1. **GPTQ**：后训练量化，4bit/8bit
2. **AWQ**：Activation-aware Weight Quantization，比 GPTQ 精度损失更小，现在更受欢迎
3. **SqueezeLLM**：更精细的二阶量化
4. **FP8**：NVIDIA Hopper 架构原生支持
5. **KV 缓存量化**：可以只量化 KV cache，不量化权重，性价比很高

---

### 5. vLLM 支持分布式推理吗？怎么分配？

**Q:** 模型太大一张卡放不下，vLLM 怎么处理？

**A:**

vLLM 支持 **张量并行（tensor parallel）**：
- 把模型权重按张量维度切分到多张 GPU 卡
- 比如 attention 头切到不同卡，MLP 也切
- 推理时每张卡算一部分，然后汇总结果
- 支持多卡推理更大模型

也支持 **paged attention 跨卡**，所以还是能保持高显存利用率。

## TensorRT 篇

### 6. 什么是 TensorRT？它做了哪些优化？

**Q:** TensorRT 是什么？为什么能加速推理？

**A:**

**TensorRT** 是 NVIDIA 推出的**深度学习推理优化器**，能把训练好的模型转换成优化过的推理引擎，在 NVIDIA GPU 上低延迟高吞吐量运行。

主要做了这些优化：

1. **算子融合（Kernel Fusion）**：把多个小算子融合成一个大 kernel，减少 kernel 启动开销和显存读写
2. **量化**：支持 FP16/BF16/INT8/FP8 量化，减少计算量和显存
3. **内核自动调优**：根据你的 GPU 型号和输入 shape，自动选择最优 kernel 实现
4. **常量折叠**：把常量提前计算好，推理时不用再算
5 **内存优化**：复用显存，减少不必要的拷贝
6. **动态形状优化**：支持动态输入长度，对 LLM 友好

**结果**：比原生 PyTorch 推理延迟低很多，吞吐量高很多。

---

### 7. TensorRT-LLM 是什么？和普通 TensorRT 有什么区别？

**Q:** TensorRT-LLM 特别在哪里？

**A:**

**TensorRT-LLM** 是 NVIDIA 专门针对**大语言模型**优化的 TensorRT 版本，普通 TensorRT 是通用深度学习优化。

针对 LLM 特别优化：
1. 对 Multi-Head Attention、Group-Query Attention 做了专门优化
2. 支持 Paged KV 缓存，和 vLLM 思路类似
3. 支持 In-flight batching（类似 vLLM continuous batching）
4. 支持各种量化方法（GPTQ AWQ Sparse TT 等）
5. 对最新 NVIDIA GPU（Hopper Ada Lovelace）做了专门优化

所以做 LLM 推理部署，现在都用 TensorRT-LLM，不用原始 TensorRT。

---

### 8. vLLM vs TensorRT-LLM 怎么选？

**Q:** 部署大模型，选 vLLM 还是 TensorRT-LLM？

**A:**

| 维度 | vLLM | TensorRT-LLM |
|------|------|-------------|
| **易用性** | 易用，直接加载 HuggingFace 模型，改几行代码就能用 | 编译引擎比较麻烦，对模型结构有限制，改模型需要重新编译 |
| **支持模型** | 基本所有主流模型都支持 | 主流热门模型有 example，自定义模型需要自己拼 |
| **吞吐量** | 很高 | 一般比 vLLM 更高一点 |
| **延迟** | 低 | 更低 |
| **生态** | 开源社区活跃，更新快 | NVIDIA 官方，对 NVIDIA GPU 优化最好 |

**选择建议：**
- ✅ **快速开发 / 自定义模型**：选 vLLM，省事
- ✅ **性能极致优化 / 生产环境上线**：选 TensorRT-LLM，性能更好
- ✅ **非 NVIDIA GPU（AMD/Apple）**：只能用 vLLM

## 量化优化篇

### 9. 大模型推理为什么需要量化？量化到底量化什么？

**Q:** 为什么要量化？量化权重还是量化激活还是量化 KV？

**A:**

**为什么需要量化：**
1. **省显存**：原本 FP16 每个参数 2 字节，4bit 量化就是 0.5 字节，同样显存能放更大模型
2. **提速**：量化计算（INT8/FP8）比 FP16 计算更快，因为数据搬运少，计算单元吞吐更高
3. **能在消费级 GPU 上跑大模型**：70B 模型 FP16要 140GB 显存，4bit 量化只要 35GB 左右，3090/4090 就能跑

**量化对象：**
- **权重量化**：最常用，对模型权重量化，省显存最明显
- **激活量化**：对每层激活量化，进一步省显存加速计算
- **KV 缓存量化**：只量化 KV 缓存，对精度影响很小，显存省很多，性价比很高

---

### 10. GPTQ 和 AWQ 有什么区别？各有什么优缺点？

**Q:** 都是 4bit 后训练量化，选哪个？

**A:**

| 对比项 | GPTQ | AWQ |
|--------|------|-----|
| 全称 | Generalized Post-Training Quantization | Activation-aware Weight Quantization |
| 核心思想 | 按块量化，利用 hessian 矩阵，对权重影响小的通道多量化 | 保留 幅值大的权重，量化幅值小的权重，对激活缩放更准 |
| 精度损失 | 比 AWQ 略大一点 | 比 GPTQ 更小，同样比特数精度更高 |
| 量化速度 | 较快 | 比 GPTQ 慢一点 |
| 现在生态 | 支持广泛 | 越来越受欢迎，现在开源模型很多 AWQ 版本 |

**现在趋势：** AWQ 精度更好，越来越受欢迎，同等比特数选 AWQ。

---

### 11. 什么是 FP8 量化？为什么 Hopper 架构（H100）推荐 FP8？

**Q:** FP8 比 INT8 好在哪里？

**A:**

**FP8** 是 8bit 浮点格式，NVIDIA Hopper 架构（H100）开始硬件原生支持。

**和 INT8 对比：**
- FP8 是浮点，范围更大，不容易溢出
- 精度比 INT8 高，尤其是激活量化
- H100 硬件原生支持 FP8 计算，不需要额外 quantization/dequantization 开销

**优势：**
- 精度损失比 INT8 小
- 推理速度和 INT8 差不多
- 对大模型来说，FP8 是现在推荐的量化方法，精度速度平衡最好

---

### 12. 量化会掉点，有哪些方法可以恢复精度？

**Q:** 量化后精度降了怎么补救？

**A:**

常见方法：
1. **知识蒸馏**：用大模型 FP16 当老师蒸馏量化后的学生模型
2. **微调（Q-LoRA）**：量化完权重，加个小 LoRA 微调几轮恢复精度
3. **更好的量化算法**：比如 AWQ 比 GPTQ 本身精度损失就小
4. **只量化权重不量化激活**：对精度影响小一点，显存节省也少一点

---

### 13. 什么是 KV 缓存量化？为什么要做？

**Q:** KV 缓存也需要量化吗？有什么好处？

**A:**

**KV 缓存**是推理时存每一步计算出来的 Key 和 Value，下次不用再算，节省时间。

长文本推理/高并发场景下，**KV 缓存占用的显存可能比权重还大**，所以量化 KV 缓存能省很多显存。

**做法：** 一般用 8bit 量化 KV，精度损失很小，显存能省一半，性价比很高。

现在 vLLM/TensorRT-LLM 都支持 KV 缓存量化。

---

### 14. 什么是 SVD 量化？GPTQ AWQ 和 SVD 是什么关系？

**Q:** SVD 量化思路是什么？

**A:**

**SVD（奇异值分解）** 是一种低秩逼近量化压缩方法：
- 把大权重矩阵分解成低秩矩阵相乘
- 减少参数数量，达到压缩效果
- 比如把 `m × n` 矩阵分解成 `m × k × k × n`，k 远小于 min(m,n)

**和 GPTQ/AWQ 区别：**
- GPTQ/AWQ 是**按比特压缩**，把每个参数从 16bit 压到 4bit
- SVD 是**低秩压缩**，减少参数个数
- 现在 4bit 量化比低秩压缩更流行，因为相同显存占用精度更高

## 部署架构篇

### 15. 大模型推理服务部署，怎么设计才能高并发低延迟？

**Q:** 生产环境部署推理服务，有哪些优化点？

**A:**

**架构层面优化：**
1. **动态批处理**：用 vLLM/TensorRT-LLM 的 continuous batching，提高 GPU 利用率
2. **分页 KV 缓存**：提高显存利用率，支持更大并发
3. **量化**：根据精度要求选 8bit/4bit/FP8，省显存放更多并发
4. **模型分片**：大模型多张卡张量并行
5. **CDN 缓存静态回答**：常见问题缓存，不用每次推理
6. **队列限流**：过载拒绝，避免雪崩
7. **监控**：监控延迟、吞吐量、GPU 利用率，告警

---

### 16. Ollama 是什么？适合什么场景？

**Q:** Ollama 为什么这么火？它做了什么？

**A:**

**Ollama** 是一个**方便本地运行大模型的工具**，把模型权重、环境配置都打包好了，一条命令就能跑：`ollama run qwen2:7b`

**优点：**
- 一键部署，非常简单
- 自动管理模型权重，下载、更新都帮你做好了
- 内置量化，适合本地开发测试
- 支持 Apple Silicon GPU 加速

**适合场景：**
- ✅ 本地开发测试
- ✅ 个人电脑本地跑大模型
- ❌ 不适合生产环境高并发部署（需要 vLLM/TensorRT-LLM）

---

### 17. 什么是 MIG？怎么用 MIG 提高 GPU 利用率？

**Q:** NVIDIA MIG 技术是什么？

**A:**

**MIG（Multi-Instance GPU）** 是 NVIDIA AMPERE 架构开始支持的 GPU 虚拟化技术：
- 把一张 A100/H100 切成多个独立的计算实例
- 每个实例有自己的显存、计算单元
- 可以同时跑多个小模型推理，提高 GPU 利用率

**适用场景：**
- 你的模型不大，一张卡用不完，切成多个实例跑多个副本，提高利用率
- 多租户隔离，不同用户用不同实例

## 总结速记

| 考点 | 核心结论 |
|------|---------|
| vLLM 核心创新 | PagedAttention 解决 KV 缓存碎片化，提高显存利用率 |
| PagedAttention | 借鉴操作系统分页，逻辑连续物理不连续，显存利用率 ~90% |
| Continuous Batching | 动态插入新请求，不用等整批完成，GPU 利用率更高 |
| vLLM vs TensorRT-LLM | vLLM 易用支持广，TensorRT-LLM 性能更好 |
| GPTQ vs AWQ | AWQ 精度更高，现在更受欢迎 |
| FP8 优势 | NVIDIA H100 原生支持，精度比 INT8 好 |
| KV 缓存量化 | 精度损失小，省大量显存，性价比很高 |
| 量化目的 | 省显存，提吞吐，让大模型能在消费级显卡跑 |
| TensorRT 主要优化 | 算子融合、量化、内核调优 |
| Ollama | 一键本地运行大模型，开发测试方便，不适合生产高并发 |
